## 1. 해결된 항목

### 1.1 주문/배달 단건 조회 접근제어 누락

- 상태: ✅ 해결
- 조치:
  - `GET /api/v1/orders/{id}`에 역할 제한과 소유권/역할 검증 추가
  - `GET /api/v1/deliveries/{id}`에 `STORE`, `ADMIN` 권한 제한 추가
- 비고: 주문 조회는 서비스 계층 접근 검증까지 연결됨

### 1.2 EventController 권한 미적용

- 상태: ✅ 해결
- 조치: Event API에 역할 기반 접근 제어 추가

### 1.3 `customer-summaries` 무인가

- 상태: ✅ 1차 해결
- 조치: `GET /api/v1/orders/frontend/customer-summaries`에 `STORE`, `ADMIN` 권한 제한 추가
- 잔여 과제: STORE 범위 tenant scoping은 추가 필요

### 1.4 `X-User-Id`, `X-User-Role` 헤더 스푸핑 위험

- 상태: ✅ 1차 해결
- 조치:
  - 각 서비스 RoleCheckInterceptor에서 JWT 직접 검증 추가
  - 전달 헤더와 JWT 클레임 불일치 시 차단
- 잔여 과제: Gateway 선행 헤더 제거/재주입, 내부 포트 비공개화

### 1.5 쿠키 보안 속성 미흡

- 상태: ✅ 해결
- 조치:
  - `ResponseCookie`로 발급/삭제 통일
  - `SameSite=Lax` 적용
  - `Secure=true` 기본값, local 프로파일만 `false`

### 1.6 Refresh Token 평문 저장

- 상태: ✅ 해결
- 조치: Refresh Token 저장/조회 기준을 SHA-256 해시로 전환
- 운영 영향: 기존 세션 일부는 재로그인이 필요할 수 있음

### 1.7 내부 서비스 JWT 검증 시크릿 누락

- 상태: ✅ 해결
- 조치: `docker-compose.yml`에 내부 서비스용 `TOKEN_SECRET` 전달 추가

## 2. 여전히 남아 있는 높음(High) 항목

### 2.1 내부 서비스 HTTP 포트 외부 노출

- 상태: 🔴 미해결
- 문제: 서비스 내부 JWT 검증이 추가됐더라도, 호스트에 직접 노출된 포트는 공격 표면을 계속 유지함
- 권장: Gateway와 필수 포트만 외부 공개, 나머지는 internal network로 제한

### 2.2 gRPC 서비스 간 통신 암호화/신뢰 경계 부재

- 상태: 🔴 미해결
- 문제: gRPC 채널은 개발 편의 중심 설정이며 운영용 TLS/mTLS 설계가 없음
- 권장: 운영 환경에서 gRPC TLS 또는 서비스 메시 적용

### 2.3 주문 상태 변경의 매장 소유권 검증 정교화

- 상태: 🔴 미해결
- 문제: `STORE` 사용자의 상태 변경 범위를 매장 소유 기준으로 더 엄격히 제한할 여지가 남아 있음
- 권장: 주문의 `storeId`와 현재 STORE 사용자 소유 매장 일치 여부를 명시 검증

## 4. 낮음/권장 항목

- Rate Limiting 미적용
- 비밀번호 복잡도 정책 부재
- 계정 잠금 메커니즘 부재
- 감사 로깅 미구현
- Actuator 노출 정책 점검 필요
- JWT 키 회전 전략 부재
- 배달 상태 변경의 세부 역할/소유권 모델 미정
