# 2026-03-10 API 생성 필요 목록

> 작성일: 2026-03-10
> 목적: 각 프론트 웹의 더미 데이터 구간을 실제 백엔드 API와 연결하기 위해 필요한 신규 API를 정리한다.

## 1. 현재 상태 요약

- web-admin: 사용자, 매장, 메뉴, 주문 단건, 배달 단건, 이벤트 일부만 실제 API 연동 완료
- web-shop: 인증과 메뉴 CRUD 일부만 실제 API 연동 완료
- web-customer: 로그인/회원가입만 실제 API 연동 완료
- 그 외 대시보드, 목록, 집계, 쿠폰, 리뷰, 찜, 문의, 공지 영역은 대부분 더미 데이터 기반

## 2. 우선 수정 완료 항목

- refresh 응답이 사용자 이메일/이름을 비워 반환하던 문제 수정
- web-shop 메뉴 관리 화면이 user id를 store id로 오인하던 문제 수정
- 점주 전용 내 매장 조회 API 추가: GET /api/v1/stores/me
- 주문 목록 API 추가: GET /api/v1/orders/my, GET /api/v1/orders/store/{storeId}
- 장바구니 API 추가: GET/POST/DELETE /api/v1/users/me/cart
- 찜 API 추가: GET/POST/DELETE /api/v1/users/me/favorites
- web-shop 주문 화면을 매장 주문 목록 API에 연결

## 3. web-admin API 생성 필요

### 3.1 운영 관리

- 리뷰 목록 조회 API
  - 화면: operation/reviews
  - 필요 기능: 리뷰 목록, 매장명/평점/상태 필터, 숨김 처리, 답글 등록/수정
- FAQ 관리 API
  - 화면: operation/faqs
  - 필요 기능: FAQ 목록, 등록, 수정, 삭제, 공개 여부, 카테고리 관리
- 문의 관리 API
  - 화면: operation/inquiries
  - 필요 기능: 문의 목록, 상태 변경, 답변 등록, 상세 조회
- 공지 관리 API
  - 화면: operation/notices
  - 필요 기능: 공지 목록, 등록, 수정, 삭제, 고정 여부, 카테고리

### 3.2 마케팅

- 쿠폰 관리 API
  - 화면: marketing/coupons
  - 필요 기능: 쿠폰 템플릿 목록, 발급 정책, 발급 수량, 사용량 집계
- 배너 관리 API
  - 화면: marketing/banners
  - 필요 기능: 배너 목록, 등록, 노출 순서, 활성화 여부, 클릭 통계

### 3.3 재무/감사

- 거래/정산 목록 API
  - 화면: finance/transactions
  - 필요 기능: 결제/환불/정산 목록, 상태 변경, 관리자 메모
- 감사 로그 API
  - 화면: system/audit
  - 필요 기능: 관리자 액션 이력 조회, 대상 리소스/시간/사용자 필터
- 시스템 로그 API
  - 화면: system/logs
  - 필요 기능: 서비스별 로그 조회, 레벨 필터, traceId 검색

### 3.4 관리자 대시보드

- 운영 요약 집계 API
  - 화면: home/HomeStats, home/HomeChart, home/HomeSales
  - 필요 기능: 기간별 주문 수, 매출, 신규 가입자, 장애/이벤트 카운트, 최근 주문/결제 목록

## 4. web-shop API 생성 필요

### 4.1 점주 매장 식별

- 내 매장 조회 API
  - 목적: 로그인한 STORE 계정이 관리하는 매장 id를 직접 조회
  - 제안 경로: GET /api/v1/stores/me
  - 응답: StoreDto 또는 StoreSummaryDto
  - 상태: 구현 완료

### 4.2 주문/운영

- 매장 주문 목록 API
  - 화면: orders
  - 필요 기능: 상태별 주문 목록, 검색, 실시간 또는 주기 갱신
  - 현재 경로: GET /api/v1/orders/store/{storeId}
- 매장 주문 상태 변경 API 확장
  - 화면: orders
  - 현재 order 단건 상태변경은 존재하나, 점주 화면에서 목록 기반 흐름이 필요
- 매장 리뷰 목록 API
  - 화면: communication/reviews
  - 필요 기능: 리뷰 목록, 답글 등록/수정
- 고객 메시지/공지 API
  - 화면: inbox, communication/notices, communication/customers
  - 필요 기능: 고객 대상 공지/메시지 전송, 발송 이력, 세그먼트 관리

### 4.3 점주 대시보드

- 매장 매출 집계 API
  - 화면: home/HomeStats, home/HomeChart
  - 필요 기능: 기간별 주문 수, 매출, 배달 중 건수, 신규 리뷰 수
- 최근 주문 피드 API
  - 화면: home/HomeSales
  - 필요 기능: 최근 주문 번호, 접수 시각, 상태, 결제 금액

### 4.4 점주 프로필/설정

- 점주 프로필 조회/수정 API
  - 화면: settings/profile
  - 현재 화면은 템플릿 데이터 사용
  - user profile 또는 store owner profile 정책 확정 필요

## 5. web-customer API 생성 필요

### 5.1 매장 탐색

- 매장 검색/목록 API 보강
  - 화면: 홈, 검색, 카테고리
  - 기존 /api/v1/stores 목록의 고객용 표시값 부족 문제 수정
  - 반영 필드: eta, reviewCount, deliveryFee, heroIcon, tags, bestseller, promo
  - 상태: 구현 완료
- 매장 상세 API
  - 필요 기능: 매장 기본 정보 + 메뉴 목록 + 운영 상태 + 리뷰 요약

### 5.2 장바구니/주문

- 장바구니 API
  - 화면: layouts/default 장바구니 모달
  - 필요 기능: 장바구니 조회, 추가, 수량 변경, 삭제, 비우기
  - 제안 경로: /api/v1/cart 또는 /api/v1/users/me/cart
  - 상태: 조회/추가/삭제/비우기 구현 완료
- 내 주문 목록 API
  - 화면: orders
  - 필요 기능: 사용자별 주문 목록, 상태 필터, 상세 조회
  - 상태: GET /api/v1/orders/my 구현 완료, customer 화면 연동 완료

### 5.3 고객 활동

- 찜 목록 API
  - 화면: favorites
  - 필요 기능: 찜 목록 조회, 추가, 삭제
  - 상태: 구현 완료
- 내 리뷰 목록 API
  - 화면: reviews
  - 필요 기능: 내가 작성한 리뷰 목록, 수정, 삭제
- 쿠폰 목록 API
  - 화면: benefits/coupons
  - 필요 기능: 보유 쿠폰, 사용 완료 쿠폰, 만료 상태
- 이벤트 목록 API
  - 화면: benefits/events
  - 현재 이벤트 서비스는 단건 생성/조회만 존재
  - 고객용 목록 API 필요

### 5.4 고객센터

- FAQ 공개 목록 API
  - 화면: support/faq
- 1:1 문의 등록/목록 API
  - 화면: support/inquiry

## 6. 구현 우선순위

### 1차

- GET /api/v1/stores/me
- GET /api/v1/stores/{storeId}/orders
- GET /api/v1/users/me/orders
- GET/POST/DELETE /api/v1/users/me/favorites
- GET/POST/PATCH/DELETE /api/v1/users/me/cart

> 2026-03-10 기준 1차 항목 중 stores/me, orders/my, orders/store, favorites, cart는 구현 완료

### 2차

- GET /api/v1/users/me/reviews
- GET /api/v1/users/me/coupons
- GET /api/v1/events
- GET /api/v1/faqs
- POST /api/v1/inquiries

### 3차

- 관리자 리뷰/FAQ/문의/공지/쿠폰/배너/거래/감사 로그 관리 API
