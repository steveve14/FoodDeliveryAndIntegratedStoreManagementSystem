# 03. 기술 스택

---

## ⚙️ Backend

| 구분 | 기술 | 버전 | 설명 |
|---|---|---|---|
| **Framework** | Spring Boot | 4.0.2 | 메인 백엔드 프레임워크 |
| **Language** | Java | 17 (LTS) | Eclipse Temurin JDK 권장 |
| **Build Tool** | Gradle (Groovy DSL) | 9.2.1 | 멀티 모듈 빌드 관리 |
| **Gateway** | Spring Cloud Gateway (WebFlux) | - | API 라우팅, JWT 검증, 헤더 전달 |
| **Discovery** | Netflix Eureka | - | 서비스 인스턴스 등록 및 검색 |
| **데이터 접근** | Spring Data JDBC | - | JPA 미사용 |
| **DB (Local)** | H2 in-memory | - | `local` 프로파일 시 자동 실행 |
| **DB (Prod)** | PostgreSQL | - | 운영 환경 RDBMS (포트 6000) |
| **내부 통신** | Spring gRPC | 1.0.2 GA | 공식 Spring 프로젝트 |
| **gRPC 라이브러리** | io.grpc | 1.77.1 | Spring gRPC BOM 관리 |
| **Protobuf** | - | 4.33.2 | Spring gRPC BOM 정렬 |
| **JWT** | jjwt | 0.12.6 | Jakarta EE 호환 |
| **Spring Cloud** | - | 2025.1.0 | Eureka / Gateway BOM |
| **코드 포맷** | Spotless + google-java-format | 1.19.2 | CI 포맷 강제 |
| **인프라** | Docker + Docker Compose | - | 8개 서비스 + PostgreSQL + Redis |

> [!note] ORM 미사용
> `spring.jpa.*` 설정 미사용. DDL 자동 생성(`ddl-auto`) 없이
> `schema.sql` / `data.sql` 스크립트로 테이블을 명시적으로 정의합니다.
