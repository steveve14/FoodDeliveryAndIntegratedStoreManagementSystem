# 01. 프로젝트 개요

> [!info] 프로젝트 목표
> 음식 배달 앱과 매장 내 POS 시스템을 **통합**하여, 사장님이 단일 대시보드에서 온/오프라인 주문을 한 번에 관리할 수 있는 플랫폼.
> 최신 기술 스택(Spring Boot MSA, Nuxt.js)을 도입하여 확장성과 유지보수성 확보.

---

## 🎯 핵심 가치 제안

- **통합 주문 관리**: 배달 앱 주문 + 매장 테이블 오더를 하나의 인터페이스에서 처리
- **실시간 동기화**: 메뉴 품절 처리 시 배달 앱·키오스크 동시 반영
- **역할 기반 접근**: 고객 / 사장님 / 직원 / 라이더 / 관리자로 분리된 권한 체계

---

## 📅 프로젝트 마일스톤

| Phase | 내용 | 상태 |
|---|---|---|
| Phase 1 | 기획/설계 - 요구사항 분석, MSA 아키텍처 설계, DB 모델링 | ✅ 완료 |
| Phase 2 | 백엔드 코어 - Eureka, Gateway, 공통 모듈, 인증 서비스 | ✅ 완료 |
| Phase 3 | 비즈니스 로직 - 주문, 가게, 결제 마이크로서비스 | ✅ 완료 |
| Phase 4 | 프론트엔드 - 사장님 대시보드, 매장 태블릿, 고객 앱 | 🔄 진행 중 |
| Phase 5 | 통합/배포 - 서비스 연동 테스트, CI/CD 구축 | 📋 예정 |

---

## 👥 역할과 책임 (R&R)

### 백엔드 (Spring Boot)

| 영역 | 담당 내용 |
|---|---|
| MSA 환경 구축 | `service-discovery`, `service-gateway` 및 `service-*` 모듈 설정/관리 |
| API 개발 | 도메인별 독립 REST API 서버 개발 |
| 데이터 설계 | Spring Data JDBC 기반 DB 접근 계층. 로컬(H2) / 운영(PostgreSQL) 분리 |
| 내부 통신 | **Spring gRPC 1.0.2 GA** - Protobuf 스키마 기반 서비스 간 통신 |
| 인증 | jjwt 0.12.6 - JWT Access/Refresh Token 발급 및 검증 |

### 프론트엔드 (Nuxt.js)

| 앱 | 설명 |
|---|---|
| `web-user` | 고객용 웹 (배달 주문) |
| `web-shop` | 사장님 대시보드 |
| `web-admin` | 관리자 시스템 |

### 모바일 (Android)

| 앱 | 패키지 | 대상 |
|---|---|---|
| `app-android-shop` | `com.fooddelivery.shop` | 사장님(매장)용 |
| `app-android-user` | `com.fooddelivery.user` | 고객용 |
| `app-android-kiosk` | `com.fooddelivery.kiosk` | 키오스크용 |
| `app-android-delivery` | `com.fooddelivery.delivery` | 라이더/배달용 |

> [!note] Android 상태
> 기본 MainActivity 스캐폴드 완료. 백엔드 API 연동은 진행 중.
> Android SDK 34, minSdk 24, Gradle 8.1.1
