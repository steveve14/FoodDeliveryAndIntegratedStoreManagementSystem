# 04. 마이크로서비스 목록

---

## 🗂️ 서비스 구성 개요

```
backend/msa-root/
├── service-discovery   # Eureka 서버
├── service-gateway     # API Gateway (WebFlux)
├── service-auth        # 인증 / JWT 발급
├── service-user        # 사용자 관리
├── service-store       # 매장 / 메뉴 관리
├── service-order       # 주문 처리
├── service-delivery    # 배달 관리
└── service-event       # 이벤트 / 알림
```

---

## 🔍 서비스별 상세

### 🌐 service-discovery
- **역할**: Netflix Eureka 서버 - 모든 서비스의 인스턴스 등록 및 검색
- **포트**: `8100`
- **실행**: 가장 먼저 기동 필요

### 🚪 service-gateway
- **역할**: 단일 진입점 (API Gateway)
  - JWT 검증 후 `X-User-Id`, `X-User-Role` 헤더를 내부 서비스에 전달
  - CORS 설정 (개발: 3010, 3100, 3200 허용)
- **포트**: `8000`
- **DB**: 없음
- **특징**: Spring Cloud Gateway (WebFlux 기반, 비동기 처리)

### 🔐 service-auth
- **역할**: 회원가입 / 로그인 / JWT 발급 / Refresh Token 관리
- **포트**: HTTP `8090`, gRPC `9090`
- **DB**: `db_user` (refresh_tokens 테이블)
- **API 주요 경로**: `/api/v1/auth/**`
- **gRPC 제공**: `ValidateToken`
- **특징**:
  - Access Token + Refresh Token (이중 토큰)
  - Refresh Token은 **SHA-256 해시**로 DB 저장
  - 쿠키 `Secure=true`, `SameSite=Lax` (`local` 프로파일에서만 `Secure=false`)

### 👤 service-user
- **역할**: 사용자 정보 CRUD, 주소록, 장바구니, 즐겨찾기
- **포트**: HTTP `8080`, gRPC `9080`
- **DB**: `db_user` (users, addresses, cart_items, favorite_stores, user_notifications)
- **API 주요 경로**: `/api/v1/users/**`
- **gRPC 제공**: `GetUserByEmail`, `Authenticate`
- **권한**: `USER`, `ADMIN`

### 🏪 service-store
- **역할**: 매장 등록/관리, 메뉴 관리, 카테고리 처리
- **포트**: HTTP `8082`, gRPC `9082`
- **DB**: `db_store` (store, menu)
- **API 주요 경로**: `/api/v1/stores/**`
- **gRPC 제공**: `GetProductById`
- **공개 API**: `GET /api/v1/stores`, `GET /api/v1/stores/*`, `GET /api/v1/stores/*/menus`
- **카테고리 지원**: `KOREAN`, `CHICKEN`, `CHINESE`, `JAPANESE`, `WESTERN`, `PIZZA`, `BURGER`, `SNACK`, `NIGHT`, `BOSSAM`, `ASIAN`, `SALAD`, `LUNCHBOX`, `CAFE` 등

### 📦 service-order
- **역할**: 주문 생성 / 조회 / 상태 전이
- **포트**: HTTP `8083`, gRPC `9083`
- **DB**: `db_order` (orders, order_item)
- **API 주요 경로**: `/api/v1/orders/**`
- **gRPC 제공**: `GetOrderById`
- **gRPC 소비**: `GetProductById` (service-store)
- **주문 상태 흐름**:
  ```
  PENDING → ACCEPTED → COOKING → DELIVERING → COMPLETED
                    └→ CANCELLED
  ```

### 🚴 service-delivery
- **역할**: 배달 배차 / 픽업 완료 / 배달 완료 처리
- **포트**: HTTP `8084`, gRPC `9084`
- **DB**: `db_delivery` (delivery)
- **API 주요 경로**: `/api/v1/deliveries/**`
- **gRPC 소비**: `GetOrderById` (service-order)
- **환경 변수**: `DELIVERY_DEFAULT_FEE` (기본값: 3000원)

### 📣 service-event
- **역할**: 이벤트 / 쿠폰 / 알림 관리
- **포트**: HTTP `8085`, gRPC `9085`
- **DB**: `db_event` (events)
- **API 주요 경로**: `/api/v1/events/**`
- **권한**: 이벤트 생성/조회 역할 기반 접근 제어 적용

---

## 🔒 공통 보안 설정 (전 서비스 적용)

> [!important] 내부 JWT 재검증
> 모든 내부 서비스(`user`, `store`, `order`, `delivery`, `event`)는
> `Authorization` 헤더 또는 `access-token` 쿠키에서 JWT를 **직접 검증**합니다.
> `X-User-Id`, `X-User-Role` 헤더와 토큰 클레임이 일치하지 않으면 요청을 차단합니다.

필요 환경 변수: `TOKEN_SECRET` (전 서비스 공유)
