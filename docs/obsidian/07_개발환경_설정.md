# 07. 개발 환경 설정

> **최종 수정**: 2026-04-05

---

## 📋 필수 요구 사항

| 도구 | 버전 | 비고 |
|---|---|---|
| **Java** | JDK 17 LTS | Eclipse Temurin 권장 |
| **Node.js** | v20 LTS 이상 | - |
| **pnpm** | v10.26.1 이상 | `npm install -g pnpm` |
| **IDE - Backend** | IntelliJ IDEA | - |
| **IDE - Frontend** | VS Code | - |
| **Git** | 최신 버전 | - |

> [!warning] pnpm 필수
> 프론트엔드 패키지 관리자는 **pnpm**을 사용합니다.
> `npm install`은 사용하지 않습니다.

---

## 🔑 필수 환경 변수

| 변수 | 용도 | 비고 |
|---|---|---|
| `TOKEN_SECRET` | Gateway + 내부 서비스 JWT 검증 | **전 서비스 동일 값 필수** |
| `DB_USER` | PostgreSQL 계정 | 운영/도커 기준 |
| `DB_PASSWORD` | PostgreSQL 비밀번호 | 운영/도커 기준 |
| `GOOGLE_CLIENT_ID` | Google OAuth 설정 | 실제 사용 시 권장 |
| `DELIVERY_DEFAULT_FEE` | 기본 배달비 | 기본값: 3000 |
| `ORDER_VIP_THRESHOLD` | VIP 주문 기준 횟수 | 기본값: 10 |

> [!danger] TOKEN_SECRET 미설정 시
> Gateway뿐 아니라 내부 서비스의 **직접 JWT 재검증도 실패**합니다.
> 전 서비스가 동일한 `TOKEN_SECRET` 값을 사용해야 합니다.

### 로컬 환경 변수 로드

```powershell
# Windows (PowerShell)
.\env-load.ps1

# Windows (cmd)
env-load.cmd
```

## 🚀 백엔드 실행 순서

MSA 특성상 **순서**가 중요합니다.

```bash
# 1. Eureka Discovery
./gradlew :service-discovery:bootRun
# → http://localhost:8100

# 2. API Gateway
./gradlew :service-gateway:bootRun
# → http://localhost:8000

# 3. Auth 서비스 (gRPC 서버 포함)
./gradlew :service-auth:bootRun

# 4. 나머지 서비스 (순서 무관)
./gradlew :service-user:bootRun
./gradlew :service-store:bootRun
./gradlew :service-order:bootRun
./gradlew :service-delivery:bootRun
./gradlew :service-event:bootRun
```

> [!info] H2 in-memory DB
> `local` 프로파일 사용 시 별도 DB 서버 불필요.
> `schema.sql` + `data.sql`이 서비스 시작 시 자동 실행됩니다.

## 🐳 Docker Compose 실행

```bash
cd backend/msa-root

# 환경 변수 설정 후
docker-compose up -d
```

> [!warning] Docker 사용 전 확인사항
> 1. `TOKEN_SECRET` 포함 환경 변수 준비 필수
> 2. 내부 서비스도 JWT 직접 검증 → auth 서비스만 설정으로는 부족
> 3. 현재 Compose는 개발 편의상 내부 서비스 포트를 외부에 노출 (운영 환경 그대로 사용 금지)

### Docker Compose 구성

```
서비스: 8개 (discovery, gateway, auth, user, store, order, delivery, event)
DB: PostgreSQL
캐시: Redis
```

## 🔐 쿠키 / 인증 주의사항

| 프로파일 | Secure | SameSite |
|---|---|---|
| `local` | false | Lax |
| 기본 / prod | true | Lax |

> [!tip] 로컬 로그인 안 될 때
> auth 서비스가 `local` 프로파일로 실행 중인지 확인하세요.
> HTTP 환경에서는 `Secure=false`가 필요합니다.
