# 13. 프론트엔드 구성

---

## 🖥️ 웹 앱 구성 (Nuxt.js)

### 개요

| 앱 | 포트 | 대상 사용자 | 패키지 경로 |
|---|---|---|---|
| `web-admin` | 3000 | 관리자 | `frontend/web-admin/` |
| `web-shop` | 3100 | 사장님 | `frontend/web-shop/` |
| `web-user` | 3200 | 고객 | `frontend/web-user/` |

---

### web-admin (관리자 대시보드)

**목적**: 플랫폼 전체 운영 관리

**주요 화면**:
- 사용자 관리 (목록, 권한 변경)
- 매장 관리 (승인, 정지)
- 주문/배달 현황 전체 조회
- 매출 통계 / 지표 대시보드
- 시스템 버전 관리

**기술 특징**:
- `useAuth` composable 기반 인증 상태 관리
- 관리자(`ADMIN`) 역할 전용 라우트 보호

---

### web-shop (사장님 대시보드)

**목적**: 개별 매장 운영 관리

**주요 화면**:
- 통합 주문 목록 (배달 + 매장 테이블 오더)
- 매장 레이아웃 설정 (드래그 앤 드롭)
- 메뉴 관리 (품절 처리, 가격 수정)
- 매출 분석 (일별/월별)

**기술 특징**:
- `useHomeStoreSource` composable로 주문 데이터 관리
- 레이아웃: `layouts/default.vue`

---

### web-user (고객 웹)

**목적**: 배달 주문 플로우

**주요 화면**:
- 홈 (카테고리 필터, 매장 목록)
- 매장 상세 (메뉴 목록)
- 장바구니 → 결제 → 주문 확정
- 주문 내역 / 상태 추적

**기술 특징**:
- `useApi`, `useAuth` composable
- 쿠키 기반 세션 관리 (`user-session`)
- SSR 쿠키 버그 수정: `if (import.meta.client)` 가드

**E2E 테스트 (Playwright)**:
- `run/e2e/order-flow.spec.ts`
- 14단계 시나리오 `1 passed (4.1s)` (2026-03-13)
- 홈 → 카테고리 필터 → 매장 상세 → 장바구니 담기 → 결제 → 주문 내역 진입

## 📱 Android 앱 구성

### 공통 사항

| 항목 | 값 |
|---|---|
| Android SDK | 34 |
| minSdk | 24 |
| Gradle | 8.1.1 |
| UI | Material Design 1.10.0, XML 레이아웃 |
| Layout | ConstraintLayout |

---

### app-android-shop (사장님용)

- **패키지**: `com.fooddelivery.shop`
- **경로**: `frontend/app-android-shop/`
- **주요 기능**: 매장 주문 관리, 상태 변경
- **API**: 주문 목록/상태 변경 연동

---

### app-android-user (고객용)

- **패키지**: `com.fooddelivery.user`
- **경로**: `frontend/app-android-user/`
- **주요 기능**: 배달 주문 생성, 상태 추적
- **API**: 매장 목록, 주문 생성/조회

---

### app-android-kiosk (키오스크용)

- **패키지**: `com.fooddelivery.kiosk`
- **경로**: `frontend/app-android-kiosk/`
- **주요 기능**: 테이블 오더 (결제 없이 주방 전송)
- **설정**: `KioskConfig` (SharedPreferences) — 스토어 ID 관리

> [!note] 키오스크 설정
> 하드코딩된 스토어 ID → `KioskConfig`(SharedPreferences 기반)로 교체 완료 (2026-03-30)

---

### app-android-delivery (라이더/배달용)

- **패키지**: `com.fooddelivery.delivery`
- **경로**: `frontend/app-android-delivery/`
- **주요 기능**: 배달 배차 수락, 픽업 완료, 배달 완료
- **API**: `/api/v1/deliveries` 연동, `DeliveryDto` 모델

## 🛠️ 프론트엔드 공통 설정

### 환경 변수 (웹)

```bash
NUXT_PUBLIC_API_BASE=http://localhost:8000
```

### pnpm 워크스페이스

```
frontend/
├── web-admin/    # pnpm dev → :3000
├── web-shop/     # pnpm dev → :3100
└── web-user/     # pnpm dev → :3200
```

### 공통 composable 패턴

```typescript
// useApi - HTTP 통신
const { data } = await useApi('/api/v1/stores')

// useAuth - 인증 상태 관리
const { user, login, logout } = useAuth()
```
