# 08. 배포 전략

---

## 🐳 Docker 빌드 전략

멀티 스테이지 빌드로 경량 이미지를 생성합니다.
`ARG SERVICE_NAME`으로 공용 Dockerfile 사용 (서비스마다 별도 파일 불필요).

```dockerfile
# Build Stage
FROM gradle:7.6-jdk17-alpine AS builder
WORKDIR /build
COPY . .
RUN ./gradlew build -x test --parallel

# Run Stage
FROM openjdk:17-jdk-slim
WORKDIR /app
ARG SERVICE_NAME
COPY --from=builder /build/${SERVICE_NAME}/build/libs/*.jar app.jar

ENV TZ=Asia/Seoul

ENTRYPOINT ["java", "-jar", \
  "-Djava.security.egd=file:/dev/./urandom", \
  "-Dspring.profiles.active=prod", \
  "app.jar"]
```

**Dockerfile 위치**: `backend/msa-root/Dockerfile`

### 환경 변수 전달 (중요)

```yaml

  TOKEN_SECRET: ${TOKEN_SECRET}
  DB_USER: ${DB_USER}
  DB_PASSWORD: ${DB_PASSWORD}
```

> [!warning] TOKEN_SECRET 전달 필수
> 내부 서비스 JWT 재검증을 위해 모든 서비스에 `TOKEN_SECRET`을 주입해야 합니다.

## ☸️ Kubernetes 리소스 구조

### Deployment 예시

```yaml
apiVersion: apps/v1
kind: Deployment
metadata:
  name: service-store
  namespace: food-delivery
spec:
  replicas: 2
  selector:
    matchLabels:
      app: service-store
  template:
    spec:
      containers:
        - name: service-store
          image: my-registry/service-store:latest
          ports:
            - containerPort: 8080
          readinessProbe:
            httpGet:
              path: /actuator/health/readiness
              port: 8080
            initialDelaySeconds: 20
          livenessProbe:
            httpGet:
              path: /actuator/health/liveness
              port: 8080
```

### Service 타입 전략

| 서비스 | K8s Service 타입 | 이유 |
|---|---|---|
| service-gateway | LoadBalancer / NodePort | 외부 트래픽 진입점 |
| 나머지 서비스 | ClusterIP | 외부 직접 접근 차단 |

## 🌐 Ingress 전략

외부(인터넷) 접근은 **Nginx Ingress Controller**를 통해 `service-gateway`로만 허용합니다.

```
인터넷 → Nginx Ingress → service-gateway (ClusterIP)
                              ↓
                    내부 서비스들 (ClusterIP)
```

```yaml
# Ingress 예시
apiVersion: networking.k8s.io/v1
kind: Ingress
metadata:
  name: food-delivery-ingress
  annotations:
    nginx.ingress.kubernetes.io/ssl-redirect: "true"
spec:
  rules:
    - host: api.mydomain.com
      http:
        paths:
          - path: /
            pathType: Prefix
            backend:
              service:
                name: service-gateway
                port:
                  number: 8000
```
