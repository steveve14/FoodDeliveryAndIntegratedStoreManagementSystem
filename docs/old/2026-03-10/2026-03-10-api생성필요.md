# 2026-03-10 API 생성 필요 목록

> 작성일: 2026-03-10
> 목적: 각 프론트 웹의 더미 데이터 구간을 실제 백엔드 API와 연결하기 위해 필요한 신규 API를 정리한다.

## 0. 2026-03-10 추가 정리

- 프론트 전역 탐색 결과, web-admin이 가장 많은 업무 더미 화면을 포함하고 있으며 web-shop, web-user도 dashboard/stat/inventory 성격의 샘플 데이터가 남아 있음
- 각 프론트의 server/api 하위 4개 mock route(customers, mails, members, notifications)는 인라인 하드코딩 배열을 제거하고 공통 provider 방식으로 정리 필요
- 위 route는 현재 실제 백엔드 API가 아니라 프론트 검증용 fallback 데이터이므로 장기적으로는 삭제 대상이며, 우선순위는 실제 API 제공 후 route 제거
- 백엔드 DB 검증용 더미 데이터는 database/2026_03_10_frontend_dummy_seed.sql 기준으로 users, addresses, favorites, cart, store, menu, orders, delivery, events까지 적재 가능하도록 맞춤
- apply_all_databases.bat는 db_user 초기화 파일 경로와 db_event 경로가 틀려 있어 함께 수정 필요

- web-admin: 사용자, 매장, 메뉴, 주문 단건, 배달 단건, 이벤트 일부만 실제 API 연동 완료
- web-shop: 인증과 메뉴 CRUD 일부만 실제 API 연동 완료
- web-user: 로그인/회원가입만 실제 API 연동 완료

## 9. server/api 정리 원칙

- server/api 내부에는 배열 하드코딩을 두지 않는다
- route는 공통 provider 또는 실제 백엔드 호출만 담당한다
- 실제 백엔드 API가 제공되면 frontend server/api fallback route는 제거한다
- 프로젝트와 무관한 외부 예시 데이터(GitHub 팀원, 영어 템플릿 메일)는 재사용하지 않는다
