# 추가 도구 도입 계획 (예정)

기준일: 2026-02-25  
대상: FoodDeliveryAndIntegratedStoreManagementSystem

## 1. 목적
- 개발/테스트/운영 생산성 향상
- 장애 탐지 및 원인 파악 시간 단축
- MSA + Kafka(예정) 환경에 맞는 품질/관측성 기준 확보

## 2. 우선순위

### P0 (즉시 도입 권장)
1. **Testcontainers**
   - 목적: PostgreSQL/Kafka 통합 테스트의 재현성 확보
   - 효과: 로컬/CI 환경 차이로 발생하는 장애 감소
2. **springdoc-openapi**
   - 목적: API 문서 자동화
   - 효과: 프론트-백엔드 협업 속도 개선
3. **Micrometer + Prometheus + Grafana**
   - 목적: 지표 수집/시각화
   - 효과: 장애 감지 시간 단축
4. **Docker Compose (개발용)**
   - 목적: 로컬 통합 실행 환경 표준화
   - 효과: 온보딩 시간 단축
5. **Bruno 또는 Postman 컬렉션 관리**
   - 목적: API 테스트 시나리오 공유
   - 효과: 반복 테스트 자동화 기반 마련

### P1 (Kafka 도입 시 권장)
1. **Kafdrop 또는 AKHQ**
   - 목적: 토픽/메시지 조회 및 디버깅
2. **Schema Registry (Confluent/Apicurio)**
   - 목적: 이벤트 스키마 버전/호환성 관리
3. **OpenTelemetry + Tempo/Jaeger**
   - 목적: 분산 트레이싱

### P2 (품질/보안 고도화)
1. **SonarQube**: 정적 분석 품질 게이트
2. **OWASP Dependency-Check**: 라이브러리 취약점 점검
3. **Trivy**: 컨테이너 이미지 취약점 점검
4. **Playwright + Vitest**: 프론트 E2E/단위 테스트 체계화

## 3. 도입 순서 (권장)
1. P0-1 ~ P0-3 먼저 도입 (테스트/문서/모니터링)
2. P0-4 개발환경 표준화 후 팀 공통 사용
3. Kafka 적용 시점에 맞춰 P1 일괄 도입
4. 릴리즈 안정화 단계에서 P2 적용
